逢甲大學學藝性社團資訊網站 - 詳細需求規格書
1. 專案概述
專案目標： 更新現有逢甲社團資訊網站，優化使用者體驗。主要訴求為提升「學藝性社團」的曝光度，並提供直覺、便利的介面，協助大一新生快速了解各社團的特色與活動資訊。
核心價值： 資訊透明化、檢索便利化、管理系統化。
2. 使用者角色與權限規劃 (RBAC)
系統將使用者分為三個層級，需實作登入驗證與權限控管機制：
一般訪客（未註冊/未登入的新生或學生）：
權限：僅供瀏覽。可檢視首頁、社團詳情、活動列表與活動詳情。
可使用所有前台的搜尋與篩選功能。
社團管理員（各社團負責人或幹部）：
權限：具備特定社團的編輯權。
可進入「社團管理後台」，更新所屬社團的介紹、上傳圖片/簡報資料。
可新增、修改、刪除由該社團主辦的活動資訊。
學校管理員（課外活動指導組或系統管理員）：
權限：最高權限 (Super Admin)。
可進入「學校管理後台」，對所有社團與所有活動進行新增、修改、刪除 (CRUD) 操作。
可管理社團管理員的帳號與權限配置。
3. 前端頁面與功能需求
3.1 首頁 (Main Page)
版面設計： 採用網格佈局 (Grid Layout) 或卡片式設計 (Card UI) 來展示學藝性社團。
社團卡片：
包含：社團徽章或自訂封面圖、社團名稱。
互動效果：滑鼠移入 (Hover) 時觸發 CSS 動畫（例如：卡片浮現、顯示簡短標語或資訊疊加層）。
點擊行為：點擊卡片跳轉至該「社團詳情頁面」。
進階篩選功能： 設置搜尋/篩選列，允許使用者透過社課地點、社課時間、收費價格區間快速篩選出符合條件的社團。
近期活動展示區： 以輪播圖 (Carousel) 或列表形式，重點展示即將到來的重大活動（如：期初社員大會、新生茶會）。
3.2 社團詳情頁面 (Club Detail Page)
基本資訊： 社團介紹（支援多段落文字）、社團宣傳圖片集。
實用資訊區塊： 清楚標示收費標準、常態性社課地點與時間。
社群連結： 提供 Instagram、Facebook、Line 社群等外部連結圖示。
教材與紀錄： 專屬區塊展示「近期社課上課簡報和內容」，可支援 PDF 預覽或圖片輪播，幫助新生具體了解社課品質與教學方向。
3.3 活動總覽頁面 (Event List Page)
列表展示： 依時間順序（由近到遠）條列顯示校內所有社團活動。
多重篩選器：
發布社團： 選擇特定社團的活動。
時間區間： 選擇特定月份或日期範圍。
活動類型： 例如「迎新」、「講座」、「實作課程」、「社遊」等分類。
點擊行為： 點擊單一活動條目即可進入「活動詳情頁面」。
3.4 活動詳情頁面 (Event Detail Page)
視覺宣傳： 頂部展示活動宣傳海報或照片。
核心資訊： 詳細的活動內容說明、精確的舉辦時間、活動地點（可結合 Google Maps 嵌入或地點文字描述）。
4. 後台管理系統需求
4.1 社團管理員專屬後台
儀表板： 登入後僅會看到自己所屬的社團資訊。
社團資料編輯器： 提供表單讓幹部修改介紹文字、上傳封面圖與上課簡報檔案。
活動管理模組： 可填寫表單建立新活動（需包含活動名稱、時間、地點、描述、宣傳圖），並能隨時進行修改或下架。
4.2 學校管理員專屬後台
全局資料表： 以表格 (Table) 形式列出所有社團與所有活動。
資料維護： 具備完整的新增 (Create)、讀取 (Read)、更新 (Update)、刪除 (Delete) 權限。可強制修改違規內容或協助遺失帳號的社團更新資訊。
5. 技術架構與開發規範
前端 (Frontend):
使用 HTML、CSS、JavaScript 進行開發。
建議可將首頁動畫與互動效果封裝，並透過 AJAX/Fetch 向後端請求 JSON 格式的資料，實現非同步渲染，提升載入體驗。
後端 (Backend):
Express.js (文檔中的 Myexpress)：建立 RESTful API 路由，處理前端發送的 GET（查詢）、POST（新增）、PUT（修改）、DELETE 請求。
實作中介軟體 (Middleware) 來處理會話 (Session) 或 Token 驗證，確保管理後台的 API 只有具備對應權限的角色才能存取。
資料庫 (Database):
SQLite：輕量級關聯式資料庫。
建議進行正規化設計，規劃如 Users (使用者表)、Clubs (社團表)、Events (活動表) 等關聯表。並將預設的 SQL 結構檔妥善保留，方便測試時快速匯入還原資料庫狀態。
